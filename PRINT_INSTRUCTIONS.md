# Instructions for Printing Code with Annotations

## Overview

All source code files are already fully annotated with comments. Follow these instructions to print them for submission.

## Files to Print

Print the following files (they are already annotated):

1. **lexical_analyzer.py** - Lexical analyzer with DFA implementation
2. **syntax_analyzer.py** - Syntax analyzer with recursive descent parser
3. **semantic_analyzer.py** - Semantic analyzer with symbol table

## Printing Options

### Option 1: Direct Print from IDE/Editor
1. Open each file in your code editor
2. File → Print
3. Select "Print syntax highlighting" if available
4. Print with line numbers (recommended)

### Option 2: Generate PDF
1. Use a tool like `pygmentize` or online syntax highlighter
2. Convert to PDF
3. Print the PDF

### Option 3: Print as Plain Text
1. Open file in text editor
2. File → Print
3. Use monospace font (Courier, Consolas, etc.)
4. Print with line numbers

## Formatting Tips

1. **Use landscape orientation** for wide code (optional)
2. **Print with line numbers** - helps reference specific lines
3. **Use readable font size** - at least 10pt
4. **Print double-sided** to save paper (optional)
5. **Add page numbers** if printing multiple pages

## Verification Checklist

Before printing, verify:
- [ ] All files have docstrings
- [ ] Complex logic has inline comments
- [ ] Algorithm explanations are present
- [ ] Error handling is documented
- [ ] Code is properly formatted

## Example Print Settings

**Recommended Settings:**
- Font: Courier New or Consolas (monospace)
- Size: 10-12pt
- Line numbers: Yes
- Syntax highlighting: Yes (if available)
- Margins: Normal (0.75" or 2cm)
- Orientation: Portrait (or Landscape for wide code)
